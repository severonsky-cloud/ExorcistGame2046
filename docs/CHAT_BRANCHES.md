# Разделение разработки между чатами

## Основной принцип

Сейчас игра всё ещё собрана в крупный `index.html`. Чтобы несколько чатов не уничтожали изменения друг друга:

1. Только интеграционный чат редактирует итоговый `index.html`.
2. Остальные чаты работают в своей ветке и своей области.
3. Каждый чат отдаёт:
   - отдельный модуль или патч;
   - короткий `HANDOFF.md`;
   - перечень изменённых систем;
   - сценарий проверки;
   - известные ограничения.
4. Никаких массовых «улучшений всего проекта» внутри одной тематической ветки.

## Ветки

### `main`

Публичный стабильный билд, который можно показывать людям.

Запрещено напрямую складывать сюда недотестированные экспериментальные системы.

### `integration/playable`

**Чат-интегратор.**

Отвечает за:

- объединение изменений из остальных веток;
- устранение конфликтов;
- smoke-тест полного игрового цикла;
- выпуск новой версии `index.html`;
- контроль, чтобы соло-режим не ломался из-за кооператива.

Это единственная ветка, где разрешена широкая правка итогового `index.html`.

### `feature/coop-netcode`

**Чат сетевого кооператива.**

Отвечает за:

- WebRTC DataChannel;
- комнаты и подключение;
- синхронизацию игроков;
- host authority;
- реконнект;
- общий квестовый state;
- синхронизацию NPC, лута и боя;
- будущий signalling/TURN слой.

Не меняет дизайн квартир, внешний вид города или боевую анимацию без отдельного согласования.

### `feature/exorcism-combat`

**Чат боёвки и сущностей.**

Отвечает за:

- атаки Санька и Прасковьи;
- ритуальные последовательности;
- реакции сущностей;
- telegraph / impact / stagger;
- модели Кровавой графини и других сущностей;
- совместные комбо двух экзорцистов;
- понятность боя для игрока.

Сетевую передачу событий описывает через публичный интерфейс, но не переписывает WebRTC.

### `feature/world-sim`

**Чат живого Ростова.**

Отвечает за:

- планировку центра;
- NPC и патрули;
- гопоту, школоту, гражданских и беженцев;
- процедурные события;
- транспорт;
- день и ночь;
- квестовые точки;
- оптимизацию количества агентов.

### `feature/interiors-inventory`

**Чат интерьеров, предметов и лута.**

Отвечает за:

- квартиры Санька и Прасковьи;
- общий подъезд;
- базу ФБК;
- инвентарь;
- контейнеры;
- артефакты;
- торговлю;
- спонсорские подарки;
- общий и личный склад.

### `feature/ui-phone-social`

**Чат UI/UX, телефона и медиа.**

Отвечает за:

- HUD;
- 2ГИС;
- телефон в руках;
- пальцы и контекстные жесты;
- селфи;
- стримы;
- канал ФБК 2046;
- комментарии, подписчиков, спонсоров;
- читаемые кооперативные индикаторы.

### `qa/playable-build`

**Чат тестирования.**

Не добавляет новые фичи. Проверяет:

- запуск на Windows, Android и настольном Chromium;
- прохождение квеста Кровавой графини;
- выход из квартиры после квеста;
- два игрока в одном секторе;
- два игрока в разных секторах;
- общий фонд;
- стримы и спонсоров;
- вызов фанатов;
- отключение второго игрока;
- сохранность соло-режима.

## Контракт между ветками

Любая новая система должна предоставлять небольшой API через `window.Exorcist2046` или отдельный модуль:

```js
window.Exorcist2046 = {
  events,
  player,
  inventory,
  quests,
  coop,
  world,
  ui
};
```

Сетевой чат передаёт игровые события, но не решает сам, как выглядит атака. Боевой чат создаёт события атаки, но не решает сам, как они отправляются по сети.

Пример:

```js
Exorcist2046.events.emit("ritual:tool", {
  playerId,
  tool: "prayer_book",
  targetId: "blood_countess"
});
```

## Правило передачи результата

В конце работы каждый чат должен оставить файл `HANDOFF.md` со структурой:

```md
# Handoff

## Что сделано
## Какие файлы изменены
## Публичные события/API
## Как проверить
## Что не работает
## Что должен сделать интеграционный чат
```
