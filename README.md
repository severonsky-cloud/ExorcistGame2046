# ExorcistGame2046

Играбельный браузерный 3D-прототип кооперативного immersive sim про экзорцистов Санька и Прасковью в разделённом Ростове 2046 года.

## Что уже работает

- настоящий WebGL/Canvas 3D-срез от первого лица;
- разные квартиры героев: аскетичная квартира Санька и квартира-склад Прасковьи;
- инвентарь, лутинг, артефакты, телефон, штатив, селфи и внутренняя соцсеть;
- квест «Кровавая графиня» с читаемым ритуалом;
- процедурный центр города, патрули и уличные события;
- **двухпользовательский P2P-кооператив через WebRTC**;
- общий подъезд: Санёк живёт на втором этаже, Прасковья — на третьем;
- общий ФБК — Фонд по борьбе с крипосущностями;
- общая касса, канал «ФБК 2046», спонсоры и фанатская поддержка;
- игроки могут находиться в одной локации либо расходиться по разным секторам;
- позиции, герои, активные инструменты, стримы и часть квестового состояния синхронизируются;
- в бою с Кровавой графиней клиент может отправлять ритуальные действия хосту.

## Как запустить

Не открывайте `index.html` двойным кликом для сетевой игры. Запустите локальный HTTP-сервер:

### Windows

```bat
serve.bat
```

Или вручную:

```powershell
py -m http.server 8080
```

Затем откройте:

```text
http://localhost:8080
```

### Linux / macOS

```bash
./serve.sh
```

## Как подключить второго игрока

1. Оба игрока открывают игру через localhost или HTTPS.
2. Каждый выбирает своего героя. Лучше выбрать разных: Санёк и Прасковья.
3. Нажать `K` для открытия кооперативной панели.
4. Игрок 1 выбирает **HOST** и нажимает **СОЗДАТЬ / ПРИНЯТЬ**.
5. Игрок 1 отправляет верхний код игроку 2.
6. Игрок 2 выбирает **JOIN**, вставляет код в верхнее поле и нажимает **СОЗДАТЬ / ПРИНЯТЬ**.
7. Игрок 2 отправляет нижний ответный код игроку 1.
8. Игрок 1 вставляет ответ в нижнее поле и нажимает **ПРИМЕНИТЬ ОТВЕТ**.
9. После прямого соединения оба увидят статус соседа.

Матчмейкинга и центрального сервера пока нет: соединение создаётся вручную, напрямую между двумя браузерами.

## Управление

- `WASD` — движение;
- мышь — обзор;
- `E` — взаимодействие;
- `M` / `P` — телефон и 2ГИС;
- `I` — инвентарь;
- `1` — молитвенник;
- `2` — артефакт;
- `3` — ритуальный раствор;
- `4` / `L` — фонарик;
- `V` — селфи;
- `B` — начать или остановить стрим;
- `K` — кооператив, касса ФБК и канал;
- `Shift` — бег;
- `C` — присесть.

## Кооперативная экономика

У каждого героя остаются личные деньги. Через кассу в общем подъезде или панель `K` можно делать взносы в ФБК.

Общими являются:

- касса ФБК;
- заработок канала;
- число подписчиков;
- спонсоры;
- заряды фанатской поддержки;
- количество закрытых заказов.

Стримы дают дополнительные деньги и подписчиков. На определённых порогах приходят спонсоры с подарками. Заряд поддержки позволяет вызвать группу фанатов, которая появляется рядом и отпугивает агрессивную гопоту.

## Текущие ограничения кооператива

Это первая играбельная P2P-основа, а не законченный сетевой релиз:

- соединение создаётся кодами вручную;
- рассчитано на двух игроков;
- хост является главным для общей экономики и совместного ритуала;
- большинство процедурных NPC пока симулируются локально на каждой машине;
- полноценная авторитетная синхронизация всех врагов, урона и лута ещё требует отдельной ветки;
- при строгом NAT WebRTC может не соединиться без TURN-сервера.

## Структура разработки

Распределение веток и готовые задания для отдельных чатов находятся в:

- [`docs/CHAT_BRANCHES.md`](docs/CHAT_BRANCHES.md)
- [`docs/CHAT_PROMPTS.md`](docs/CHAT_PROMPTS.md)
- [`docs/COOP_ARCHITECTURE.md`](docs/COOP_ARCHITECTURE.md)

`main` должен оставаться демонстрационным играбельным билдом. Параллельные чаты не должны одновременно переписывать `index.html`: они готовят модули и отчёты в своих областях, а интеграционный чат переносит проверенные изменения в сборку.
